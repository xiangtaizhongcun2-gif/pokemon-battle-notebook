# Pokémon Battle Notebook

ポケモンのパーティ構築と対戦の振り返りを、ひとつの場所で管理するシンプルなWebアプリです。

## 主な機能

- パーティの作成・編集・削除
- 1パーティにつき最大6匹の登録
- 全国図鑑と姿違いの検索
- メガシンカ、リージョンフォーム、ロトムなどの個別登録
- タイプ、弱点、耐性、無効、種族値の表示
- 特性、持ち物、性格、テラスタイプ、技4つの記録
- 努力値の入力と合計510チェック
- 構築コンセプト、基本選出、苦手な相手のメモ
- 対戦履歴の追加・編集・削除
- LocalStorageへの自動保存

## UI方針

画面は次の3つだけです。

1. パーティ
2. ポケモン図鑑
3. 対戦履歴

白背景、黒とグレーを中心に、装飾や画面遷移を最小限にしています。Notionの複雑なページ機能は採用せず、余白と読みやすさだけを参考にしています。

## 図鑑データ

全国図鑑と姿違いのデータは、PokéAPIの公開CSVから取得します。

- 全国図鑑の通常種
- メガシンカ
- リージョンフォーム
- ロトム、デオキシスなどのフォルム違い
- バトル中だけ変化する姿
- 見た目だけが異なる姿

日本語名、英語名、タイプ、特性、種族値を読み込みます。既存のローカル図鑑は、通信できない状態で初めて起動した場合の予備データとして残しています。

タイプ相性は特性による変更を考慮せず、選択した姿のタイプから計算します。

## オフライン利用

初回のみインターネットへ接続して図鑑データを読み込む必要があります。読み込みに成功すると、姿違いを含む全図鑑データがブラウザのLocalStorageへ保存されます。

その後はインターネット接続がなくても、同じPC・同じブラウザで図鑑検索、パーティ編集、対戦履歴を利用できます。キャッシュが7日を過ぎた場合はオンライン時に更新を試みますが、通信に失敗しても保存済みデータは削除せず、そのまま利用します。

アプリを動かすローカル開発サーバーは起動しておく必要があります。

## 開発環境

- Vite
- React
- TypeScript
- LocalStorage

## 起動方法

```bash
npm install
npm run dev
```

## 確認コマンド

```bash
npm run build
npm run lint
npm run format:check
```

## 対象外

最初のバージョンでは、次の機能は実装しません。

- ダメージ計算
- ログイン
- クラウド同期
- パーティ共有
- ターン単位の詳細な対戦ログ
- Notion風ブロックエディター
